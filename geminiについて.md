## ⚠️ 【重要】モデルの選択と使い分け（2026年1月時点）

<aside>
⚡

**研修を始める前に必ず確認！**

Geminiのモデル名称と中身が変更されました。

**業務品質を維持するため、画面上部のメニューが「Pro」になっていることを確認してください。**

</aside>

### 新旧の名称と中身の差分

**【これまで】**

- **思考モード(Pro)** ＝ Gemini 2 Pro（最高性能）
- **高速モード** ＝ 2.5 Flash（軽量版）

**【これから】**

- **Pro** ＝ Gemini 2 Pro（最高性能）
- **思考モード** ＝ Gemini 2 Flash（軽量版：思考ステップ）
- **高速モード** ＝ Gemini 2 Flash（軽量版）

<aside>
💡

**これまで「思考モード」を使っていた業務には、今後は「Pro」の利用が推奨です。**

</aside>

### 推奨使い分け

新しい「2 Flash」モデルも短時間あたりの性能は上がっていますが、**アウトプット全体の精度は「Pro」が勝ります。**

基本は「**Pro**」を使いつつ、タスクに応じて「高速モード」などを活用してください。

**● Pro（★推奨 / メイン業務）**

専門知識の広さ、複雑な指示への追従、文脈の深い理解に優れます。最終的なアウトプットの精度が最も安定しています。

- **用途：**企画、資料作成、専門調査、コード生成、壁打ち、相談、意思決定の判断材料等
- **理由：**これが旧「思考モード（2 Pro）」の正統な後継設定です。最も知識の裾野が広く、高い品質が保証されています。

**● 高速モード（作業効率化）**

中身は最新の「Gemini 2 Flash」です。レスポンスが非常に早く、初速のスピード感に優れています。

- **用途：**翻訳、長文の要約、メールの下書き整形、ブレストの初期出しなど
- **理由：**質より速度を優先する作業に適しています。業務内容によっては積極的に活用することで効率化が可能です。

**● 思考モード（特殊：論理整理用）**

回答の前に「思考ステップを書き出す」のが特徴で理屈に強いモードです。思考する時間がかかるため回答速度は遅くなります。

- **用途：**複雑なロジックの整理、デバッグ方針の検討、Proの回答に違和感がある時
- **理由：**中身は軽量モデル（Flash）に思考プロセスを足したものです。深い専門知識を必要とする相談事にはProが向いていますが、論理的な導出プロセスを確認したい時に真価を発揮します。

## 🎓 Step 1：基本操作（20分）

### 1-1. まずは話しかけてみる（5分）

**やってみよう：**

```
「こんにちは。あなたは何ができますか？」
```

**次はこう聞いてみる：**

```
「マーケティング施策のアイデアを5つ教えて」
「Python初心者におすすめの学習ステップを教えて」
```

**ポイント：**

- 完璧なプロンプトは不要。まずは普通に話しかけてOK
- 質問→回答→追加質問、の対話で精度を上げていく

### 1-2. メール文章を直す（15分）

**やってみよう：**

```
以下のメールをもっと丁寧でビジネスライクな表現に修正してください。

---
お疲れ様です。
先日の件ですが、来週の打ち合わせの日程調整お願いします。
よろしくです。
```

**発展：こんな使い方もできる**

- 「このメールを英語に翻訳して」
- 「お詫びのトーンで書き直して」
- 「箇条書きで要点をまとめて」

**実務での活用シーン：**

- 取引先へのお詫びメール作成
- 社内報告メールの下書き
- 顧客への丁寧な返信文

## 📝 Step 2：業務文書作成（30分）

### 2-1. 議事録を作成する（15分）

**やってみよう：**

音声入力やメモから議事録を自動生成

```
以下の会議メモから議事録を作成してください。
形式：日時、参加者、議題、決定事項、次のアクション、の5項目で整理

---
（会議メモを貼り付け）
昨日の営業会議で、新商品Aの販売戦略について話した。
田中さんが価格設定は競合より10%安くすべきと提案。
佐藤さんは品質訴求で勝負したいとのこと。
結論：来週までに両案で試算を出す。担当は田中さん。
```

**ポイント：**

- 雑なメモでもOK。Geminiが構造化してくれる
- 「もっと詳しく」「簡潔に」で調整可能

**発展：Google Meetの自動議事録を精度アップ**

- Google Meetの自動議事録をコピー
- Step 3-2で作成した「議事録作成Gem」に貼り付け
- → 会社のコンテキストを踏まえた、読みやすい議事録に自動変換

### 2-2. 報告書・提案書の下書き（15分）

**やってみよう：**

```
以下のテーマで週次報告書を作成してください。
構成：今週の実績、課題、来週の計画、の3部構成

テーマ：営業活動報告
実績：新規5件訪問、既存フォロー10件、成約2件
課題：A社の契約が保留中
来週：B社との商談、C社フォロー
```

**発展：こんな使い方もできる**

- 「この報告書に競合分析セクションを追加して」
- 「エグゼクティブサマリーを冒頭に追加して」

## 🧠 Step 3：コンテクストを入れた高度な使い方（30分）

### 3-1. 会社情報を含めて精度を上げる（15分）

**基本形（精度：低）**

```
「新商品のキャッチコピーを考えて」
```

→ 汎用的な回答しか返ってこない

**コンテクスト入り（精度：高）**

```
あなたは請求管理SaaSのマーケターです。
当社の商品「〇〇」は、〇〇（ターゲット）の請求業務を自動化するシステムです。
ターゲットは50代で、紙やExcelでの作業に疲弊しています。
このターゲットに刺さるキャッチコピーを5案作成してください。
```

**ポイント：**

- 役割（マーケター）
- 商品情報（請求業務自動化）
- ターゲット（50代、紙/Excel疲れ）

を明示すると、実用的な回答が得られる

### 3-2. コンテキストをGeminiに保存する（10分）

**重要：毎回同じ情報を入力するのは面倒**

→ Geminiの「Gem」機能で、会社情報や自分の役割を保存できます

**やってみよう：**

1. Geminiの画面右上の「Gem Manager」をクリック
2. 「新しいGemを作成」を選択
3. 以下のような情報を保存：

```jsx
【Gem名】営業サポートAI

【指示内容】
あなたは当社の営業担当者のアシスタントです。

【会社情報】
- 事業内容：（ここに自社の事業内容を記載）
  例：BtoB向けクラウドサービスの提供
- 主要サービス/製品：（主力商品を記載）
  例：業務効率化ツール「〇〇」
- ターゲット顧客：（主な顧客層を記載）
  例：中小企業の経営者・管理職
- 競合との差別化：（自社の強みを記載）
  例：導入後のサポート体制が手厚い

【あなたの役割】
営業担当として、以下の業務をサポート：
- 提案書・見積書の作成
- 顧客への説明資料の準備
- 商談後のフォローメール作成
- 営業活動の記録・報告書作成

【出力の方針】
- ビジネス文書は丁寧で分かりやすく
- 専門用語は必ず説明を添える
- 顧客の課題解決を最優先に考える
- 社内向けは簡潔に、社外向けは丁寧に
```

**メリット：**

- 毎回会社情報を入力する手間がゼロに
- チャットを新しく始めても、常に同じコンテキストで回答
- 複数のGemを作って用途別に使い分け可能（営業用、企画用、分析用など）

**実務活用例：**

- 営業用Gem：顧客対応、提案書作成
- マーケ用Gem：広告コピー、分析レポート
- 人事用Gem：採用資料、評価シート
- **議事録作成用Gem：会議の内容を網羅的に記録**

**【重要】議事録作成用Gemのすすめ**

<aside>
📝

**Google Meetの自動議事録だけでは不十分な理由**

Googleワークスペースを使っていれば、Google Meetで自動的に議事録が作成されます。

しかし、これには**企業情報や議事録のフォーマット指定が入っていません**。

→ より精度の高い議事録を作るために、専用のGemを作成するのがおすすめです。

</aside>

**議事録作成用Gemの例：**

```jsx
【Gem名】議事録作成AI

【指示内容】
あなたは当社の会議議事録を作成する専門アシスタントです。
会議に参加しなかった人でも、議事録を読むだけで会議の内容を完全に理解できる、網羅的で分かりやすい議事録を作成してください。

【会社の方向性・戦略】
- 事業方針：（自社の事業方針を記載）
  例：顧客満足度を最優先に、持続的な成長を目指す
- 今期の重点戦略：（現在の重点施策を記載）
  例：新規事業の立ち上げ、既存事業の効率化
- 組織文化：（意思決定の文化を記載）
  例：データに基づく意思決定、迅速なPDCA

【議事録のフォーマット】
以下の構成で議事録を作成してください：

## 1. サマリー（会議の全体像）
- 会議の目的
- 主な決定事項（箇条書き、3-5項目）
- 重要なアクションアイテム（担当者・期限付き）

## 2. 決定事項の詳細
各決定事項について：
- 何を決めたのか
- なぜその決定に至ったのか（背景・理由）
- 誰が責任者で、いつまでに何をするのか

## 3. 議論の内容（詳細）
- 議題ごとに整理
- 誰が何を発言したか（重要な意見は発言者付きで記録）
- 出た意見の賛成・反対の論点
- 保留事項や今後の検討課題

## 4. 次回までのアクション
- タスクリスト形式（担当者・期限・内容）
- 優先順位付き

【重要な原則】
- 決定事項と議論は明確に分ける
- 専門用語には説明を添える
- 数値やデータは正確に記録
- 曖昧な表現は避け、具体的に記述
- 会議に参加していない人が読んでも「何が決まって、誰が何をするのか」が一目で分かる構成にする

【除外すべき内容】
- 雑談や本題と関係ない会話
- 重複する発言（要点をまとめる）
```

**このGemを使うメリット：**

- Google Meetの自動議事録をこのGemに投げるだけで、構造化された読みやすい議事録に変換
- 会社の方向性を踏まえた議事録になるため、文脈が理解しやすい
- 「誰が何をいつまでにやるか」が明確になり、アクション漏れを防止
- 新メンバーや休んだ人でも、議事録だけで会議の全体像を把握可能

### 3-3. フォーマット指定で欲しい形に（5分）

**やってみよう：**

```
以下のデータから週次レポートを作成してください。

【フォーマット】
1. 先週比の数値変化（表形式）
2. 変化の原因（箇条書き）
3. 次週の打ち手（箇条書き）

【データ】
売上：先週100万円 → 今週120万円
訪問件数：先週20件 → 今週25件
成約率：先週10% → 今週12%
```

**発展パターン：**

- 「Markdown形式で出力」
- 「スプレッドシートにコピペできる形式で」
- 「プレゼン用のスライド原稿として」

## 🔗 Step 4：Googleドライブ連携（30分）

### 4-1. スプレッドシートを読み込む（15分）

**やってみよう：**

1. Geminiのチャット画面で「📎」アイコンをクリック
2. 「Googleドライブ」を選択
3. 分析したいスプレッドシートを選択
4. プロンプトを入力：

```
このスプレッドシートのデータを分析して、以下を教えてください：
1. 売上の傾向（月別）
2. 最も売れている商品Top3
3. 異常値や気になる点
```

**メリット：**

- ダウンロード→アップロードの手間がゼロ
- リアルタイムデータで即座に分析
- 週次ルーチンで「毎週金曜15時に先週データを分析」が可能

### 4-2. Googleドキュメントを要約する（15分）

**やってみよう：**

1. Googleドライブから長文の議事録や報告書を選択
2. プロンプト：

```
このドキュメントを以下の形式で要約してください：
- 3行サマリー
- 重要な決定事項（箇条書き）
- 次のアクション（担当者付き）
```

**実務活用：**

- 100ページの契約書を2ページに要約
- 1時間の会議議事録を5分で読める形に
- 過去の提案書から再利用できる部分を抽出

**【実践ワークフロー】議事録をスプレッドシートで管理する**

<aside>
✨

**議事録作成後のベストプラクティス**

1. 議事録作成Gemで網羅的な議事録を作成
2. 作成した議事録をGoogleスプレッドシートに貼り付けて保存
3. 次回以降、そのスプレッドシートをGeminiに読み込ませる
4. 「前回会議で決まったことを基に、今週のタスクを整理して」で即座に実務に落とせる
</aside>

**具体例：**

**Step 1：議事録をスプレッドシートに保存**

- Googleスプレッドシートで「会議議事録管理」シートを作成
- 列：日付、会議名、決定事項、担当者、期限、ステータス
- Gemで作成した議事録の「決定事項」と「アクション」をコピー＆ペースト

**Step 2：作業時にスプレッドシートをコンテキストに**

1. Geminiで「📎」アイコンをクリック
2. 「会議議事録管理」スプレッドシートを選択
3. プロンプト：

```jsx
この議事録の決定事項を基に、今週私がやるべきタスクを優先順位付きでリストアップしてください。
```

```jsx
前回会議で決まった「新商品Aの価格設定」について、試算資料のドラフトを作成してください。
```

**メリット：**

- 会議で決まったことを忘れずに実行できる
- 複数の会議の決定事項を一元管理
- 「あの会議で何決まったっけ？」が瞬時に検索可能
- タスク漏れを防止し、期限管理が容易に

## 📚 Step 5：NotebookLM活用（30分）

### 5-1. 社内ナレッジを知識ベース化（15分）

**やってみよう：**

1. NotebookLM（[https://notebooklm.google.com/）を開く](https://notebooklm.google.com/）を開く)
2. 「ソースを追加」から会社のGoogleドライブを選択
3. 複数の資料をまとめて追加：
    - 過去の提案書
    - 商談記録
    - 分析レポート
    - 製品マニュアル
    - **会議議事録のスプレッドシート（重要！）**

**プロンプト例：**

```
「過去3年で成約した案件の共通パターンは？」
「〇〇業界向けの提案で使われた訴求ポイントをリストアップ」
「この顧客タイプにはどのアプローチが効果的だった？」
```

**議事録スプレッドシートを追加するメリット：**

```jsx
「先月の会議で決まったことで、まだ実行できていないものは？」
「新商品Aに関する過去の会議でどんな決定がされた？」
「今週の会議で決めるべきことを、前回までの流れを踏まえて提案して」
```

**顧客の声を入れてインサイトを発見：**

NotebookLMに顧客アンケート、インタビュー記録、問い合わせ内容を追加すると、顧客の本音からインサイトを発見できます。

**入力するソース：**

- 顧客アンケートの回答（スプレッドシートまたはPDF）
- 顧客インタビューの文字起こし
- カスタマーサポートへの問い合わせ履歴
- 営業ヒアリングの議事録
- SNSやレビューサイトの声

**プロンプト例：**

```jsx
「顧客が最も不満に感じているポイントを、3つにまとめて」
「顧客が「期待していたがなかった機能」をリストアップ」
「競合と比較して当社が優れていると感じている点は？」
「ペルソナ別に顧客のニーズを分類して」
「この声を基に、次の製品改善の優先順位を提案して」
```

**実務での活用例：**

- 新機能開発の優先順位付け：100件の顧客インタビューから「最も要望が多い機能」を抽出
- マーケティング訴求の改善：顧客が使う「生の言葉」を見つけて、広告コピーに反映
- カスタマーサポートの改善：頻繁に出る質問を分析し、FAQやマニュアルを充実
- 競合分析：「競合から乗り換えた理由」を集約し、強みを可視化

**メリット：**

### 応用1：Google Apps Script（GAS）で定型作業を自動化

**GASとは：**

Googleワークスペース内で動くJavaScriptベースのスクリプト言語。

Gmail、スプレッドシート、ドキュメント、カレンダーなどを自動化できます。

**こんな作業が自動化できる：**

- Gmailで特定の件名のメールを受信したら、内容をスプレッドシートに自動追記
- スプレッドシートのデータを毎朝自動で集計してレポートをメール送信
- カレンダーの予定から会議議事録テンプレートを自動作成
- フォーム回答を自動で集約して分析

**例：Gmailからスプレッドシートに自動コピペ**

**今までのやり方：**

1. 毎日Gmailを開く
2. 「注文報告」という件名のメールを探す
3. メール本文から「注文番号」「金額」「顧客名」をコピー
4. スプレッドシートを開いて貼り付け
5. 毎日繰り返し、コピペ漏れも発生

**GASで自動化後：**

→ 毎朝8時に自動実行。人間が何もしなくても勝手にスプレッドシートに追記される

**GeminiにGASを書いてもらおう：**

```jsx
Gmailで件名に「注文報告」を含む未読メールを検索し、
メール本文から以下の情報を抽出して、
Googleスプラッドシートに自動で追記するGoogle Apps Scriptを作成してください。

抽出する情報：
- 注文番号
- 顧客名
- 金額
- 注文日

スプレッドシートURL：（自分のスプレッドシートURLを貼り付け）

毎日朝8時に自動実行するトリガーも設定してください。
```

→ Geminiがコードを書いてくれます！

**導入ステップ：**

1. Googleスプレッドシートを開く
2. 「拡張機能」→「Apps Script」をクリック
3. Geminiが生成したコードを貼り付け
4. 「保存」→「実行」でテスト
5. 「トリガー」で毎日自動実行を設定

**メリット：**

- プログラミング知識がなくても、Geminiがコードを書いてくれる
- 「エラーが出ました」とGeminiに伝えれば、修正もしてくれる

### プラン別の使い分け

**無料版・個人プラン：**

- ❌ 機密情報・個人情報の入力は絶対NG
- データが学習に使われる可能性あり

**エンタープライズプラン：**

- ✅ データは学習に使われない契約
- ✅ 個人情報の要約・分析も可能
- ⚠️ ただし、契約範囲を情報セキュリティ部門と確認すること

**全プラン共通の注意点：**

- Geminiの回答は「たたき台」。重要事項は必ず裏取り
- 「考えるのをやめる」のではなく「考える時間を作る」ツール

## 📊 組織展開での障壁と突破法

### 障壁1：「AIに頼ると思考力が落ちる」派

**突破法：** 「考える時間を奪う作業」vs「考えるべき本質」を分けて説明

- 議事録要約30分→5分にして、その25分で戦略を練る時間を作る

### 障壁2：「結局使わない」ROM専の量産

**突破法：** 週1回の「Gemini活用事例共有会」（5-10分）

- 営業Aさん「提案書作成を10分に短縮」→他メンバーも追随

### 障壁3：「プロンプトが分からない」で挫折

**突破法：** 社内プロンプトライブラリを構築

- 「週次レポート作成」「競合分析」「リスク洗い出し」のテンプレを共有
